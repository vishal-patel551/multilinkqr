MultiLinkQR offers two file formats for downloading your QR codes: PNG (raster) and SVG (vector). PNG files are perfect for most uses including printing on business cards, flyers, or posters. They're available in small (300x300px), medium (500x500px), and large (1000x1000px) sizes. For high-quality printing, we recommend the large size. They're ideal for professional printing, large-format uses like banners or billboards, or when you need to edit the QR code in design software like Adobe Illustrator or Figma. SVG files are also smaller in file size and perfect for web use. Choose PNG for quick printing and general use, or SVG when you need maximum flexibility and quality.

MultiLinkQR requires all links to use HTTPS (not HTTP) for several important security and compatibility reasons. HTTPS ensures that the connection between the user's device and the destination website is encrypted, protecting against man-in-the-middle attacks and data interception. Modern browsers like Chrome, Safari, and Firefox display security warnings or block HTTP sites entirely, which would create a poor user experience when someone scans your QR code. Additionally, many features like geolocation, camera access, and push notifications only work on HTTPS sites. Most modern websites already use HTTPS by default. If a website you want to link to only has HTTP, try adding "https://" instead of "http://" - many sites automatically redirect. If the site truly doesn't support HTTPS, you might want to reconsider linking to it as it may not be secure or trustworthy for your users.

If one of the websites you've linked to goes offline or changes its URL, users will see an error when they try to access that specific link. For single-link QR codes, they'll get a "site not found" error from their browser. For multi-link QR codes, the other working links will still function normally - only the broken link will fail. The QR code itself continues to work; the issue is with the destination website. This is why we recommend periodically testing your QR codes, especially those printed on permanent materials. If you discover a broken link, you have a few options: generate a new QR code with the updated URL, use a URL redirect service that you control to point to a new destination, or if the broken link is on a multi-link QR code and not critical, you might leave the code as-is and let other links continue working. For important QR codes on permanent materials, consider using URL shorteners or your own domain redirects as an intermediate step, giving you more flexibility to update destinations without changing the QR code.

The ideal QR code size depends on viewing distance and scanning conditions. As a general rule, the minimum size is 2cm x 2cm (about 0.8 inches square) for close-range scanning like business cards or menus. For posters viewed from 1-2 meters away, aim for 5-7cm square. For billboards or signs scanned from several meters away, you'll need 20cm or larger. The formula is: QR code size = scanning distance รท 10. So if people will scan from 3 meters away, make your QR code at least 30cm square. Always leave "quiet zone" (white space) around your QR code - at least 4 modules (small squares) wide on all sides. When in doubt, make it larger - bigger QR codes are easier to scan and provide a better user experience. Test print at your intended size and try scanning from the expected distance before finalizing. For variable viewing distances (like a poster that might be viewed up close or from far away), err on the larger side.

You can place QR codes on colored or textured backgrounds, but you must be very careful to ensure scannability. Troubleshooting

If your QR code won't scan, systematically check these common issues. First, verify size - is the QR code large enough for the scanning distance? Try moving closer or using a larger print. Check contrast - is there sufficient difference between the QR code and background? Low contrast makes scanning difficult. Look for the quiet zone - is there white space around all four sides of the code? Quiet zone violations cause scan failures. Inspect print quality - are the individual squares clear and sharp, or blurry and distorted? Check for damage - are there scratches, folds, or printing defects covering parts of the code? Try different angles - is glare or reflection blocking the scanner? Test lighting - too dark or too bright can prevent scanning. Try a different phone - some older devices struggle with certain QR code types. Check the camera - is it clean and focusing properly? If using custom colors or logos, try regenerating with standard black-and-white to rule out design issues. For printed codes, compare a digital version on screen - if that scans but print doesn't, the issue is print quality.
